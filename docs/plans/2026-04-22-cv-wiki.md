# CV Wiki Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Build a personal CV site as an Obsidian/Quartz wiki — skills as the spine, projects as evidence, signaltrace-site as the structural template.

**Architecture:** Two-stage repo. `raw/` holds re-runnable evidence dumps (defuddled web pages, per-project dives, PDF extracts). `content/` is the hand-written editorial vault, generated *from* raw/ but never auto-stitched. Quartz scaffolding cloned from `signaltrace-site` and rebranded.

**Tech Stack:** Quartz v4 (TypeScript), Markdown + Obsidian-flavoured wikilinks, Vercel deploy, Python/Node tooling for raw extraction (defuddle skill, Explore subagents).

**Design doc:** `docs/plans/2026-04-22-cv-wiki-design.md`

**House rules — apply to every task:**
- No "Claude" / "Anthropic" / AI-attribution anywhere in commits, code, or content.
- Git operations only when the user explicitly asks (via `ydcoza-git-actions` skill). Commit steps below are **checkpoints** — present the diff and wait for the user to trigger the commit themselves.
- DRY where genuine duplication exists; YAGNI for v1 polish (no custom plugins, no theming).

**Step 1: Dispatch one Explore subagent per project, in a single message (parallel).**

Each subagent prompt template:

```
Source path: /home/laudes/zoot/projects/<dir>

Read the codebase enough to populate a CV evidence note. I need you to extract — not invent — the following sections, formatted as markdown. Be concrete; quote the code/docs where helpful. Skip sections that genuinely don't apply.

Output frontmatter:
---
source_path: /home/laudes/zoot/projects/<dir>
extracted: 2026-04-22
---

Sections:
1. **One-line pitch** — what is this project, in plain English?
2. **Stack & dependencies** — read package.json / requirements.txt / composer.json / go.mod. Name the actual technologies.
3. **Architecture sketch** — entry points → key modules → data sinks. 5–10 lines.
4. **Design decisions worth telling** — the *why* behind non-obvious choices. Mine CLAUDE.md, AGENTS.md, README, top-level commit messages, comments. Aim for 3–5.
5. **Concrete proof points** — measurable things: LOC (use `find . -name '*.py' | xargs wc -l` style commands via Bash), age (first/last commit), schema scale, integration count, anything that survives "really?".
6. **Skills demonstrated** — which of these specialities does this project prove? Pick from: Quant Engineering, AI / Agentic Systems, WordPress & PHP Craft, Knowledge Graphs & Wiki Systems, Python Services & Data Pipelines, Design & Brand. Multi-select OK.
7. **Pull quotes** — 2–4 verbatim snippets from CLAUDE.md / README / comments that capture Michael's voice or the project's character.
8. **Open questions / dead-ends** — what would Michael do differently? What's unfinished? (Mine TODO comments, CLAUDE.md "known issues" sections, recent commit messages mentioning regret/refactor.)

Write the result to: raw/projects/<slug>.md (slug = lowercase-kebab project name).

Return a 2-line summary of what you wrote.
```

**Step 2: Wait for all 7 to return.**
